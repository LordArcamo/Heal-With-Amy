## Commands

```bash
# Development
npm run dev           # Start dev server at http://localhost:4321
npm run build         # Production build
npm run preview       # Preview production build

# Unit tests (Vitest)
npm test              # Watch mode
npm run test:run      # Single run
npm run test:coverage # Coverage report (HTML output)

# E2E tests (Playwright) — requires dev server running
npm run test:e2e      # Headless across 3 device profiles
npm run test:e2e:ui   # Interactive Playwright UI

# Storybook
npm run storybook     # Dev server at http://localhost:6006
npm run build-storybook
```

To run a single Vitest test file:
```bash
npx vitest run src/components/__tests__/ChatBot.test.tsx
```

To run a single Playwright spec:
```bash
npx playwright test tests/e2e/home.spec.ts
```

## Architecture

**Astro 4 static site** — no backend, no CMS. All page content is hardcoded in `.astro` files. React is used only for the ChatBot component (`client:load` directive).

### Routing
File-based: `src/pages/*.astro` maps directly to URLs. Pages: `/`, `/about`, `/services`, `/classes`, `/testimonials`, `/contact`, `/book`.

### Layout & Master Template
`src/layouts/Layout.astro` wraps every page. It handles:
- SEO meta tags, OpenGraph, Twitter Card, canonical URLs
- JSON-LD schema (`HealthAndBeautyBusiness`)
- Google Fonts preconnect (Cormorant Garamond + Raleway)
- Scroll-reveal animation setup (IntersectionObserver + `.reveal` class)
- ChatBot mount point

### Styling
No Tailwind — plain CSS with design tokens defined as CSS custom properties in `src/styles/global.css`. Key tokens:
- Colors: `--cream`, `--brown`, `--sage`, `--gold` (and light/dark variants)
- Typography: `--font-display` (Cormorant Garamond, headings), `--font-body` (Raleway)
- Fluid sizing via `clamp()` for type and spacing

Animations use `.reveal` (scroll-triggered) and `.fade-up-{2,3,4}` (CSS delay stagger) classes.

### ChatBot
`src/components/ChatBot.tsx` — FAQ-driven keyword matching (not LLM). Responses are hardcoded in the `FAQ_RESPONSES` array (lines 8–53). Pattern matching uses word-boundary regex. Add new FAQ entries there.

### Testing Setup
- **Vitest**: jsdom environment, globals enabled, setup file at `src/test/setup.ts` (mocks `scrollIntoView`). Coverage scoped to `src/components/**`.
- **Playwright**: 3 device projects — Desktop Chrome, Mobile Safari (iPhone 13), Tablet (iPad gen 7). Base URL `http://localhost:4321`. Playwright starts the dev server automatically via `webServer` config.

### External Integrations
- Booking page embeds **Calendly**
- Site URL: `https://www.healwithamy.com` (set in `astro.config.mjs`)
- Sitemap auto-generated by `@astrojs/sitemap`
